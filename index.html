🚗 2. Kelas Mobil - Named Constructor

🔍 Konsep: Named constructor memberikan nama berbeda untuk cara pembuatan objek yang berbeda, membuat kode lebih ekspresif.

💡 Named constructor seperti "jalur khusus" pembuatan objek - memudahkan pembacaan kode.
📁 mobil.dart DART 
class Mobil {
  String? merk;
  int? tahun;

  // Parameterized Constructor
  Mobil(this.merk, this.tahun);

  // Named Constructor: Mobil Baru
  Mobil.baru() {
    merk = 'Nissan Skyline GT-R R34';
    tahun = 2002;
  }
}

🎨 3. Kelas Warna - Constant Constructor

🔍 Konsep: Constant constructor membuat objek yang tidak berubah (immutable) dan dioptimasi memory dengan canonicalization.

💡 Objek const akan menggunakan memory yang sama jika nilainya identik.
📁 warna.dart DART 
class Warna {
  final int r, g, b;

  // Constant Constructor
  const Warna(this.r, this.g, this.b);

  @override
  String toString() => 'Warna(r: $r, g: $g, b: $b)';
}

void main() {
  const warnaMerah = Warna(255, 0, 0);
  const warnaHijau = Warna(0, 255, 0);
  
  print(warnaMerah);
}

📐 4. Kelas Titik - Initializer List

🔍 Konsep: Initializer list memungkinkan manipulasi nilai sebelum constructor body dieksekusi.

💡 Initializer list dieksekusi sebelum constructor body - cocok untuk validasi awal.
📁 titik.dart DART 
class Titik {
  final double x, y;

  // Initializer list untuk memastikan koordinat bernilai absolut
  Titik(double x, double y)
      : x = x.abs(),
        y = y.abs() {
    print('Titik (${this.x}, ${this.y}) dibuat.');
  }
}

👨‍🎓 5. Kelas Mahasiswa - Named Constructor

🔍 Konsep: Named constructor dengan berbagai cara inisialisasi: default, dari Map, dan anonim.

📁 mahasiswa.dart DART 
class Mahasiswa {
  String nama;
  String nim;
  String jurusan;
  String kelas;

  // Default Constructor
  Mahasiswa(this.nama, this.nim, this.jurusan, this.kelas);

  // Named Constructor: dari Map
  Mahasiswa.dariMap(Map<String, String> data)
      : nama = data['nama']!,
        nim = data['nim']!,
        jurusan = data['jurusan']!,
        kelas = data['kelas']!;
}

👤 6. Kelas Person - Enkapsulasi Dasar

🔍 Konsep: Enkapsulasi menyembunyikan data internal dan menyediakan akses melalui getter/setter dengan validasi.

📁 person.dart DART 
class Person {
  String _nama; // Private field
  int _umur;    // Private field

  Person(this._nama, this._umur);

  // Getter
  String get nama => _nama;
  int get umur => _umur;

  // Setter dengan validasi
  set nama(String namaBaru) {
    if (namaBaru.isNotEmpty) _nama = namaBaru;
  }
}

📦 7. Kelas Produk - Validasi pada Setter

🔍 Konsep: Validasi pada setter untuk memastikan data yang masuk sesuai aturan (nama minimal 3 karakter, harga > 0).

📁 produk.dart DART 
class Produk {
  String _nama;
  double _harga;

  Produk(this._nama, this._harga);

  set nama(String value) {
    if (value.length >= 3) _nama = value;
  }

  set harga(double value) {
    if (value > 0) _harga = value;
  }
}

🔐 8. Kelas User - Sistem Login

🔍 Konsep: Enkapsulasi dengan metode login dan ganti password yang memiliki validasi berlapis.

📁 user.dart DART 
class User {
  String _username;
  String _password;
  String _email;

  User(this._username, this._password, this._email);

  bool login(String username, String password) {
    return username == _username && password == _password;
  }

  void gantiPassword(String lama, String baru) {
    if (lama == _password && baru.length >= 6) {
      _password = baru;
    }
  }
}
